# PhishBuster RU — Реализация системы проверки по трём метрикам

## Метрика 1: Фирменные DOM-селекторы

### Описание алгоритма

Фишинговые сайты часто копируют HTML-структуру оригинального сайта, включая CSS-классы, идентификаторы форм, атрибуты элементов. Проверка DOM-селекторов позволяет обнаружить:
- Формы авторизации, скопированные с оригинала
- Логотипы и брендовые элементы
- Специфичные CSS-классы (например, `.sber-login-form`, `.passp-form`)

### Псевдокод

```javascript
/**
 * Проверяет наличие фирменных DOM-селекторов на странице
 * @param {Object} pageElements - извлечённые элементы страницы
 * @param {Object} serviceSelectors - эталонные селекторы сервиса
 * @returns {Object} { matched: boolean, matchCount: number, matchedList: string[] }
 */
function evaluateSelectors(pageSelectors, serviceSelectors) {
  const allSelectors = [
    ...(serviceSelectors.loginForms || []),
    ...(serviceSelectors.brandElements || []),
    ...(serviceSelectors.securityIndicators || [])
  ];
  
  const matchedList = [];
  
  for (const selector of allSelectors) {
    try {
      const elements = document.querySelectorAll(selector);
      if (elements.length > 0) {
        matchedList.push({
          selector: selector,
          count: elements.length
        });
      }
    } catch (e) {
      // Невалидный селектор — пропускаем
      console.warn(`Невалидный селектор: ${selector}`);
    }
  }
  
  // Порог: минимум 2 совпадения для уверенного определения
  const matched = matchedList.length >= 2;
  
  return {
    matched: matched,
    matchCount: matchedList.length,
    matchedList: matchedList,
    score: matched ? 1 : 0  // нормализованное значение
  };
}
```

### Достоинства
| Достоинство | Описание |
|---|---|
| Высокая точность | CSS-селекторы уникальны для каждого сервиса |
| Быстрота | `querySelectorAll()` — нативный метод, O(n) |
| Устойчивость к визуальной подделке | Даже если фишинг выглядит как оригинал, HTML-структура выдаёт копирование |
| Обнаружение контрафакта | Если селекторы Сбербанка найдены на домене xyz — это фишинг |

### Ограничения
| Ограничение | Описание | Митигация |
|---|---|---|
| Изменение вёрстки | Сервис может обновить HTML | Регулярное обновление rules.json |
| Динамические классы | CSS-классы могут генерироваться (CSS Modules) | Использование стабильных селекторов (form, input) |
| SPA-фреймворки | React/Vue могут рендерить DOM с задержкой | MutationObserver для повторного анализа |

## Метрика 2: Ключевые слова в title и body

### Описание алгоритма

Фишинговые сайты используют названия брендов в заголовке страницы (`<title>`) и тексте, чтобы убедить пользователя в подлинности. Проверка ключевых слов определяет, упоминается ли бренд на странице.

### Псевдокод

```javascript
/**
 * Проверяет наличие ключевых слов в title и body страницы
 * @param {string} pageTitle - содержимое <title>
 * @param {string} bodyText - текст страницы (первые 5000 символов)
 * @param {Object} serviceKeywords - эталонные ключевые слова сервиса
 * @returns {Object} { matched: boolean, matchCount: number, details: Object }
 */
function evaluateKeywords(pageTitle, bodyText, serviceKeywords) {
  const titleLower = pageTitle.toLowerCase();
  const bodyLower = bodyText.toLowerCase();
  
  // Проверка title-ключевых слов (наиболее значимые)
  const titleMatches = [];
  for (const keyword of serviceKeywords.titleKeywords) {
    if (titleLower.includes(keyword.toLowerCase())) {
      titleMatches.push(keyword);
    }
  }
  
  // Проверка body-ключевых слов
  const bodyMatches = [];
  for (const keyword of serviceKeywords.bodyKeywords) {
    if (bodyLower.includes(keyword.toLowerCase())) {
      bodyMatches.push(keyword);
    }
  }
  
  // Проверка urgency-ключевых слов (социальная инженерия)
  const urgencyMatches = [];
  if (serviceKeywords.urgencyKeywords) {
    for (const keyword of serviceKeywords.urgencyKeywords) {
      if (bodyLower.includes(keyword.toLowerCase())) {
        urgencyMatches.push(keyword);
      }
    }
  }
  
  // Логика определения совпадения
  // Совпадение в title важнее, чем в body
  const titleWeight = titleMatches.length > 0 ? 1 : 0;
  const bodyWeight = bodyMatches.length >= 2 ? 1 : 0;
  
  const matched = titleWeight === 1 || bodyWeight === 1;
  
  return {
    matched: matched,
    matchCount: titleMatches.length + bodyMatches.length,
    titleMatches: titleMatches,
    bodyMatches: bodyMatches,
    urgencyMatches: urgencyMatches,
    score: matched ? 1 : 0
  };
}
```

### Достоинства
| Достоинство | Описание |
|---|---|
| Простота реализации | Строковый поиск — базовая операция |
| Устойчивость к изменениям DOM | Работает даже если HTML полностью переработан |
| Обнаружение urgency-приёмов | Выявляет приёмы социальной инженерии |
| Языковая специфика | Ключевые слова на русском — работают для русскоязычного фишинга |

### Ограничения
| Ограничение | Описание | Митигация |
|---|---|---|
| Ложные совпадения | Слово «Сбербанк» может упоминаться в статье | Комбинация с доменной метрикой (весовая модель) |
| Регистрозависимость | Разные написания | Приведение к lowercase |
| Сокращения | «Сбер» вместо «Сбербанк» | Расширенный список keyword'ов |
| Unicode-обход | Кириллические символы, похожие на латинские | Нет в текущей версии (ограничение) |

## Метрика 3: Доменные паттерны

### Описание алгоритма

Доменное имя — самый надёжный индикатор фишинга. Алгоритм проверяет:
1. **Белый список** (legitimate domains) — если домен в списке → безопасно
2. **Подозрительные паттерны** (suspiciousPatterns) — если домен совпадает с паттерном → фишинг
3. **Нейтральный домен** — если ни то, ни другое → нет данных

### Псевдокод

```javascript
/**
 * Проверяет домен на соответствие эталонным паттернам
 * @param {string} pageDomain - домен текущей страницы (hostname)
 * @param {Object} serviceDomains - эталонные доменные данные сервиса
 * @returns {Object} { status: string, score: number, reason: string|null }
 */
function evaluateDomain(pageDomain, serviceDomains) {
  const domain = pageDomain.toLowerCase();
  
  // Шаг 1: Проверка белого списка (легитимные домены)
  for (const legitDomain of serviceDomains.legitimate) {
    if (domain === legitDomain || domain.endsWith('.' + legitDomain)) {
      return {
        status: "legitimate",
        score: 0,
        reason: null,
        details: `Домен ${domain} является легитимным`
      };
    }
  }
  
  // Шаг 2: Проверка по regex-паттернам
  for (const pattern of serviceDomains.patterns) {
    const regex = new RegExp(pattern, 'i');
    if (regex.test(domain)) {
      // Домен совпадает с паттерном легитимного домена,
      // но НЕ находится в белом списке → подозрительно
      // (Например: sberbank.ru.xyz)
      return {
        status: "suspicious",
        score: 1,
        reason: `Домен похож на ${serviceDomains.legitimate[0]}, но не является легитимным`,
        details: `Совпадение с паттерном: ${pattern}`
      };
    }
  }
  
  // Шаг 3: Проверка подозрительных паттернов
  for (const suspiciousPattern of serviceDomains.suspiciousPatterns) {
    const regex = new RegExp(suspiciousPattern, 'i');
    if (regex.test(domain)) {
      return {
        status: "phishing",
        score: 1,
        reason: `Подозрительный домен: совпадение с паттерном "${suspiciousPattern}"`,
        details: `Домен ${domain} похож на известный фишинговый паттерн`
      };
    }
  }
  
  // Шаг 4: Нейтральный домен
  return {
    status: "neutral",
    score: 0,
    reason: null,
    details: `Домен ${domain} не распознан`
  };
}
```

### Примеры работы

| Входной домен | Проверка | Результат |
|---|---|---|
| `online.sberbank.ru` | ∈ legitimate | ✅ `legitimate` |
| `sberbank-secure.xyz` | ∈ suspiciousPatterns | 🚨 `phishing` |
| `my-sber-login.tk` | ∈ suspiciousPatterns | 🚨 `phishing` |
| `habr.com` | ∉ any | ⚪ `neutral` |
| `gosuslugi.com` | совпадает с pattern но ∉ legitimate | 🚨 `suspicious` |

### Достоинства
| Достоинство | Описание |
|---|---|
| Детерминированность | Результат однозначный, нет вероятностей |
| Быстрота | Regex-проверка для 5 сервисов — доли миллисекунды |
| Белый список | Полное исключение FP для легитимных сайтов |
| Масштабируемость | Добавление нового паттерна — одна строка в JSON |

### Ограничения
| Ограничение | Описание | Митигация |
|---|---|---|
| Punycode-атаки | `xn--sberbank-...` визуально выглядит как `sberbank.ru` | Проверка punycode-доменов (в планах развития) |
| Поддомены | `sberbank.com.evil.com` — поддомен evil.com | Проверка полного домена, не только суффикса |
| Новые TLD | Появление новых доменных зон | Регулярное обновление паттернов |
| IDN-домены | Кириллические домены (`.рф`) | Включение IDN-вариантов в legitimate-список |

## Сводная таблица трёх метрик

| Метрика | Что проверяет | Надёжность | Сложность реализации | FP-риск |
|---|---|---|---|---|
| Доменные паттерны | Совпадение домена с эталонами | ⭐⭐⭐⭐⭐ | Низкая | Минимальный |
| DOM-селекторы | HTML-элементы на странице | ⭐⭐⭐⭐ | Средняя | Низкий |
| Ключевые слова | Названия брендов в тексте | ⭐⭐⭐ | Низкая | Средний |

### Почему именно три метрики?

1. **Одна метрика** — недостаточно: можно обойти одним способом
2. **Две метрики** — лучше, но есть слепые зоны
3. **Три метрики** — оптимально: каждая покрывает ограничения других
4. **Четыре и более** — усложнение без значительного прироста точности для школьного проекта
