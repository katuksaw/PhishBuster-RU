## 2. Структура тестовой фишинговой страницы

Пример HTML для тестовой страницы «Фишинг Сбербанка»:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Сбербанк Онлайн — Вход в систему</title>
  <!-- Имитация фишинга: title совпадает, но домен поддельный -->
</head>
<body>
  <div class="sber-header">
    <img alt="Сбер" src="data:image/svg+xml,...">
    <!-- Фирменные селекторы -->
  </div>
  
  <form action="/collect" method="POST" id="login-form" class="login-form">
    <input type="text" name="login" placeholder="Логин">
    <input type="password" name="password" placeholder="Пароль">
    <button type="submit" class="sber-btn">Войти</button>
  </form>
  
  <div class="alert">
    <p>СРОЧНО! Ваш аккаунт заблокирован. Немедленно подтвердите вход!</p>
    <!-- Urgency-сигналы -->
  </div>
</body>
</html>
```

## 5. Критерии успеха

| Критерий | Метрика | Целевое значение |
|---|---|---|
| Accuracy (точность) | (TP + TN) / Total | ≥ 85% |
| Precision (прецизионность) | TP / (TP + FP) | ≥ 80% |
| Recall (полнота) | TP / (TP + FN) | ≥ 80% |
| F1-score | 2 × (P × R) / (P + R) | ≥ 80% |
| Время анализа | Среднее время | < 1 сек |
| FP Rate | FP / (FP + TN) | ≤ 10% |

### Ожидаемые результаты для 20 тестов

При правильной реализации:
- **TP** (True Positive) ≈ 6 — фишинг выявлен как фишинг
- **TN** (True Negative) ≈ 12 — легитимные сайты определены как безопасные/нейтральные
- **FP** (False Positive) ≈ 0-1 — легитимный сайт ошибочно помечен как фишинг
- **FN** (False Negative) ≈ 0-1 — фишинг не обнаружен

**Ожидаемая Accuracy**: 18/20 = 90%
**Ожидаемый F1-score**: ~90%
