# PhishBuster RU — Динамический контент и кэширование

## Часть A: Система обработки динамического контента (MutationObserver)

### Проблема

Современные веб-приложения (SPA — Single Page Applications) на React, Vue, Angular загружают контент **динамически**:
- DOM строится не при загрузке страницы, а через JavaScript
- Формы авторизации появляются через 1-3 секунды после `document_idle`
- Content script может не обнаружить селекторы, которые ещё не отрендерены

**Пример**: Страница ВКонтакте (`vk.com`) может загрузить форму входа через React только через 2 секунды после первоначальной загрузки DOM.

### Решение: MutationObserver

`MutationObserver` — это браузерный API, который позволяет **наблюдать за изменениями DOM** и реагировать на них.

```javascript
// content.js — система MutationObserver

let analysisTimeout = null;
const DEBOUNCE_DELAY = 500; // мс — задержка перед повторным анализом

/**
 * Инициализирует MutationObserver для отслеживания изменений DOM
 */
function initDynamicAnalysis() {
  const observer = new MutationObserver((mutationsList) => {
    // Фильтрация значимых изменений
    let hasSignificantChange = false;
    
    for (const mutation of mutationsList) {
      // Нас интересуют только добавленные узлы
      if (mutation.type === 'childList' && mutation.addedNodes.length > 0) {
        for (const node of mutation.addedNodes) {
          // Проверяем, что добавлен элемент (не текстовый узел)
          if (node.nodeType === Node.ELEMENT_NODE) {
            // Проверяем, что добавленный элемент содержит значимые теги
            const significantTags = ['FORM', 'INPUT', 'BUTTON', 'IMG', 'A', 'DIV'];
            if (significantTags.includes(node.tagName) || 
                node.querySelector('form, input, button, img, a')) {
              hasSignificantChange = true;
              break;
            }
          }
        }
      }
      if (hasSignificantChange) break;
    }
    
    if (hasSignificantChange) {
      // Debounce — защита от избыточных пересчётов
      if (analysisTimeout) {
        clearTimeout(analysisTimeout);
      }
      
      analysisTimeout = setTimeout(() => {
        console.log('[PhishBuster] DOM изменён — повторный анализ');
        runAnalysis();
      }, DEBOUNCE_DELAY);
    }
  });
  
  // Наблюдение за всем body с поддеревом
  observer.observe(document.body, {
    childList: true,     // Добавление/удаление элементов
    subtree: true        // Наблюдение за всеми потомками
  });
  
  return observer;
}
```

### Защита от избыточных пересчётов (Debounce)

**Проблема**: SPA-приложение может изменять DOM сотни раз за секунду (анимации, виртуальный скроллинг). Без защиты расширение будет запускать анализ на каждое изменение.

**Решение**: **Debounce** — техника, при которой функция выполняется только если с момента последнего вызова прошло достаточно времени.

```
Без debounce:
  DOM change → analyze (200ms)
  DOM change → analyze (210ms)
  DOM change → analyze (215ms)
  DOM change → analyze (220ms)
  → 4 анализа за 20 мс — избыточно!

С debounce (500ms):
  DOM change → [ждём 500ms]
  DOM change → [перезапускаем таймер]
  DOM change → [перезапускаем таймер]
  DOM change → [перезапускаем таймер]
  → 500ms без изменений → 1 анализ
  → 1 анализ вместо 4!
```

**Почему 500 мс**:
- 100 мс — слишком мало, SPA-сайты делают частые обновления
- 1000 мс — слишком долго, пользователь может успеть ввести данные
- **500 мс** — баланс: достаточно для «успокоения» DOM и достаточно быстро для безопасности

### Полная интеграция в content.js

```javascript
// content.js — полный модуль

let lastAnalysisResult = null;
let observer = null;

// Инициализация при загрузке
document.addEventListener('DOMContentLoaded', () => {
  // Первый анализ после загрузки
  setTimeout(() => {
    runAnalysis();
    // Запуск MutationObserver после первого анализа
    observer = initDynamicAnalysis();
  }, 1000); // Даём SPA время на первичный рендер
});

// Если DOM уже загружен (document_idle может сработать после DOMContentLoaded)
if (document.readyState === 'complete' || document.readyState === 'interactive') {
  setTimeout(() => {
    runAnalysis();
    observer = initDynamicAnalysis();
  }, 1000);
}

/**
 * Основная функция анализа страницы
 */
function runAnalysis() {
  const metrics = extractMetrics();
  
  chrome.runtime.sendMessage(
    { action: "analyzePage", data: metrics },
    (response) => {
      if (response) {
        lastAnalysisResult = response;
        displayResult(response);
      }
    }
  );
}

/**
 * Извлечение метрик из DOM
 */
function extractMetrics() {
  const forms = document.querySelectorAll('form');
  const passwordInputs = document.querySelectorAll('input[type="password"]');
  const allInputs = document.querySelectorAll('input');
  const images = document.querySelectorAll('img');
  const buttons = document.querySelectorAll('button, input[type="submit"]');
  const links = document.querySelectorAll('a');
  
  return {
    url: window.location.href,
    domain: window.location.hostname,
    title: document.title,
    bodyText: document.body ? document.body.innerText.substring(0, 5000) : "",
    timestamp: Date.now(),
    selectors: {
      forms: Array.from(forms).map(f => ({
        action: f.action || '',
        method: f.method || '',
        id: f.id || '',
        classes: f.className || ''
      })),
      passwordFields: passwordInputs.length,
      inputs: allInputs.length,
      images: Array.from(images).slice(0, 20).map(img => ({
        alt: img.alt || '',
        src: img.src ? img.src.substring(0, 200) : ''
      })),
      buttons: buttons.length,
      links: links.length
    }
  };
}

// Остановка наблюдателя при уходе со страницы
window.addEventListener('beforeunload', () => {
  if (observer) {
    observer.disconnect();
  }
});
```

## Часть B: Механизм кэширования

### Зачем нужно кэширование

**Проблема**: Popup запрашивает результат анализа при каждом открытии. Без кэша:
- background.js заново загружает rules.json
- content.js заново анализирует DOM
- Пользователь ждёт 200-800 мс каждый раз

**Решение**: Сохранить результат анализа в `chrome.storage.local` при первом вычислении, затем читать из кэша.

### Реализация

```javascript
// background.js — модуль кэширования

const CACHE_TTL = 3600000; // 1 час в миллисекундах

/**
 * Сохраняет результат анализа в кэш
 * @param {number} tabId - ID вкладки
 * @param {string} url - URL страницы
 * @param {Object} result - результат анализа
 */
async function saveToCache(tabId, url, result) {
  const cacheKey = `cache_${tabId}`;
  
  const cacheEntry = {
    url: url,
    result: result,
    timestamp: Date.now(),
    expiresAt: Date.now() + CACHE_TTL
  };
  
  await chrome.storage.local.set({ [cacheKey]: cacheEntry });
}

/**
 * Получает результат из кэша
 * @param {number} tabId - ID вкладки
 * @param {string} currentUrl - текущий URL страницы
 * @returns {Object|null} - кэшированный результат или null
 */
async function getFromCache(tabId, currentUrl) {
  const cacheKey = `cache_${tabId}`;
  
  const data = await chrome.storage.local.get(cacheKey);
  const cached = data[cacheKey];
  
  if (!cached) return null;
  
  // Проверка срока действия
  if (Date.now() > cached.expiresAt) {
    await chrome.storage.local.remove(cacheKey);
    return null;
  }
  
  // Проверка, что URL не изменился (пользователь мог перейти на другую страницу)
  if (cached.url !== currentUrl) {
    return null;
  }
  
  return cached.result;
}

/**
 * Очищает весь кэш
 */
async function clearCache() {
  const allData = await chrome.storage.local.get(null);
  const cacheKeys = Object.keys(allData).filter(key => key.startsWith('cache_'));
  await chrome.storage.local.remove(cacheKeys);
}

/**
 * Очищает просроченные записи кэша
 */
async function cleanExpiredCache() {
  const allData = await chrome.storage.local.get(null);
  const now = Date.now();
  const expiredKeys = [];
  
  for (const [key, value] of Object.entries(allData)) {
    if (key.startsWith('cache_') && value.expiresAt && now > value.expiresAt) {
      expiredKeys.push(key);
    }
  }
  
  if (expiredKeys.length > 0) {
    await chrome.storage.local.remove(expiredKeys);
    console.log(`[PhishBuster] Очищено ${expiredKeys.length} просроченных записей`);
  }
}
```

### Интеграция в background.js

```javascript
// background.js — обработчик сообщений

chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
  
  if (message.action === "analyzePage") {
    handleAnalyzePage(sender.tab.id, message.data)
      .then(sendResponse)
      .catch(error => sendResponse({ error: error.message }));
    return true; // Важно! Указывает, что sendResponse будет вызван асинхронно
  }
  
  if (message.action === "getResult") {
    handleGetResult(message.tabId)
      .then(sendResponse)
      .catch(error => sendResponse({ error: error.message }));
    return true;
  }
});

async function handleAnalyzePage(tabId, metrics) {
  // Проверяем кэш
  const cached = await getFromCache(tabId, metrics.url);
  if (cached) {
    console.log(`[PhishBuster] Результат из кэша для ${metrics.url}`);
    return cached;
  }
  
  // Выполняем анализ
  const result = calculateRisk(metrics, rulesDatabase);
  
  // Сохраняем в кэш
  await saveToCache(tabId, metrics.url, result);
  
  return result;
}

async function handleGetResult(tabId) {
  // Получаем URL активной вкладки
  const tab = await chrome.tabs.get(tabId);
  return await getFromCache(tabId, tab.url);
}
```

### Структура хранения в chrome.storage.local

```
chrome.storage.local:
├── cache_1          → { url, result, timestamp, expiresAt }    # вкладка 1
├── cache_2          → { url, result, timestamp, expiresAt }    # вкладка 2
├── cache_5          → { url, result, timestamp, expiresAt }    # вкладка 5
└── settings         → { theme: "auto", notifications: true }   # настройки (опционально)
```

### Ключи хранения

| Ключ | Формат | Назначение |
|---|---|---|
| `cache_{tabId}` | JSON-объект | Результат анализа для конкретной вкладки |
| `settings` | JSON-объект | Пользовательские настройки (опционально) |

### Обоснование CACHE_TTL = 1 час

| Значение TTL | Обоснование |
|---|---|
| 5 минут | Слишком мало — при активном серфинге постоянные пересчёты |
| 1 час | **Оптимально**: страница обычно не меняется в течение сессии |
| 24 часа | Слишком долго — база правил может обновиться |
| Бесконечно | Недопустимо — устаревшие данные |

### Почему chrome.storage.local, а не localStorage

| Хранилище | Доступ из SW | Доступ из content.js | Общий для расширения |
|---|---|---|---|
| `chrome.storage.local` | ✅ Да | ✅ Да (с permissions) | ✅ Да |
| `localStorage` | ❌ Нет (SW не имеет DOM) | ⚠️ Только на своей странице | ❌ Нет |
| `sessionStorage` | ❌ Нет | ⚠️ Только в своей вкладке | ❌ Нет |
| `IndexedDB` | ✅ Да | ✅ Да | ✅ Да |

**Почему chrome.storage.local**: Простой API, достаточно для хранения кэша, доступен из всех контекстов расширения. IndexedDB избыточен для данного проекта.
