# PhishBuster RU — Логическая схема работы расширения

## Пошаговый алгоритм работы

### Этап 1: Загрузка страницы пользователем

```
Пользователь вводит URL → Chrome начинает загрузку → 
  DOM начинает строиться → Chrome проверяет manifest.json → 
    Chrome внедряет content.js (run_at: "document_idle")
```

**Что происходит**: Когда DOM полностью загружен (`document_idle`), Chrome автоматически внедряет `content.js` в контекст страницы.

**Почему `document_idle`**: 
- `document_start` — DOM ещё не построен, селекторы не найдутся
- `document_end` — DOM построен, но изображения/стили могут ещё загружаться
- `document_idle` — **оптимально**: DOM готов, ресурсы загружены, пользователь ещё не успел взаимодействовать

### Этап 2: Инициализация content.js

```javascript
// content.js — начало работы

// Шаг 2.1: Извлечение базовых метрик
const pageUrl = window.location.href;
const pageDomain = window.location.hostname;
const pageTitle = document.title;

// Шаг 2.2: Извлечение DOM-селекторов
// Ищем характерные элементы: формы, логотипы, ссылки, кнопки входа
const forms = document.querySelectorAll('form');
const inputs = document.querySelectorAll('input[type="password"]');
const links = document.querySelectorAll('a[href]');
const images = document.querySelectorAll('img[src]');
const buttons = document.querySelectorAll('button, input[type="submit"]');

// Шаг 2.3: Сбор текстового содержимого
const bodyText = document.body.innerText.substring(0, 5000);

// Шаг 2.4: Формирование объекта метрик
const metrics = {
  url: pageUrl,
  domain: pageDomain,
  title: pageTitle,
  forms: forms.length,
  hasPasswordFields: inputs.length > 0,
  selectors: extractSelectors(forms, inputs, buttons),
  keywords: extractKeywords(pageTitle, bodyText),
  timestamp: Date.now()
};
```

### Этап 3: Отправка метрик в background.js

```javascript
// content.js → background.js

chrome.runtime.sendMessage(
  { action: "analyzePage", data: metrics },
  (response) => {
    if (response) {
      displayResult(response);
    }
  }
);
```

**Почему через sendMessage, а не напрямую**:
- Content script и service worker работают в **разных контекстах выполнения**
- В Manifest V3 у content script ограничен доступ к `chrome.*` API
- Service worker имеет доступ к `chrome.storage` и файлам расширения (rules.json)
- Это архитектурное ограничение браузера, а не дизайнерское решение

### Этап 4: Загрузка базы правил в background.js

```javascript
// background.js — инициализация

let rulesDatabase = null;

async function loadRules() {
  try {
    const response = await fetch(chrome.runtime.getURL('rules.json'));
    rulesDatabase = await response.json();
    console.log(`База правил загружена: версия ${rulesDatabase.version}, ${rulesDatabase.services.length} сервисов`);
  } catch (error) {
    console.error('Ошибка загрузки rules.json:', error);
    // Fallback: пустая база
    rulesDatabase = { version: "0.0.0", services: [] };
  }
}

// Загрузка при старте service worker
loadRules();
```

**Когда загружается**: 
- При первом запуске расширения
- При «пробуждении» service worker (после периода неактивности)
- Service worker в MV3 может быть «усыплён» и «пробуждён» — данные нужно загружать заново

### Этап 5: Вычисление итогового риска (скоринг)

```javascript
// background.js — ядро скоринга

function calculateRisk(metrics, rules) {
  let totalScore = 0;
  let matchedService = null;
  const reasons = [];

  // Шаг 5.1: Определение целевого сервиса
  // Ищем, на какой легитимный сайт похожа текущая страница
  for (const service of rules.services) {
    const matchResult = matchService(metrics, service);
    
    if (matchResult.confidence > 0.3) {
      // Похоже на этот сервис — проверяем, легитимный ли домен
      matchedService = service;
      
      // Шаг 5.2: Проверка домена
      const domainScore = evaluateDomain(metrics.domain, service);
      totalScore += domainScore.score;
      if (domainScore.reason) reasons.push(domainScore.reason);
      
      // Шаг 5.3: Проверка селекторов
      const selectorScore = evaluateSelectors(metrics.selectors, service);
      totalScore += selectorScore.score;
      if (selectorScore.reason) reasons.push(selectorScore.reason);
      
      // Шаг 5.4: Проверка ключевых слов
      const keywordScore = evaluateKeywords(metrics.keywords, service);
      totalScore += keywordScore.score;
      if (keywordScore.reason) reasons.push(keywordScore.reason);
      
      break; // Анализируем только первое совпадение
    }
  }

  // Шаг 5.5: Если не похож ни на один сервис — нейтральный результат
  if (!matchedService) {
    return {
      score: 0,
      verdict: "neutral",
      reasons: ["Сайт не похож на известные сервисы"],
      serviceName: null,
      rulesVersion: rules.version
    };
  }

  // Шаг 5.6: Определение вердикта
  const verdict = determineVerdict(totalScore);

  return {
    score: Math.min(totalScore, 100),
    verdict: verdict,
    reasons: reasons,
    serviceName: matchedService.name,
    rulesVersion: rules.version
  };
}
```

### Этап 6: Определение вердикта

```javascript
function determineVerdict(score) {
  if (score >= 60) return "danger";   // Красный — фишинг!
  if (score >= 30) return "warning";  // Жёлтый — подозрительно
  return "safe";                       // Зелёный — безопасно
}
```

**Логика порогов**:
| Score | Вердикт | Цвет | Интерпретация |
|---|---|---|---|
| 0–29 | safe | 🟢 Зелёный | Сайт легитимный или не похож на фишинг |
| 30–59 | warning | 🟡 Жёлтый | Есть подозрительные признаки, нужна осторожность |
| 60–100 | danger | 🔴 Красный | Высокая вероятность фишинга |

**Почему такие пороги**:
- `60` для danger: два сильных признака (домен + селекторы) должны сработать для уверенного выявления
- `30` для warning: один признак может быть случайным — не пугаем ложными тревогами

### Этап 7: Кэширование результата

```javascript
// background.js — сохранение в кэш

async function cacheResult(tabId, result) {
  const cacheKey = `result_${tabId}`;
  const cacheEntry = {
    score: result.score,
    verdict: result.verdict,
    reasons: result.reasons,
    serviceName: result.serviceName,
    rulesVersion: result.rulesVersion,
    url: result.url,
    timestamp: Date.now()
  };

  await chrome.storage.local.set({ [cacheKey]: cacheEntry });
}
```

### Этап 8: Отображение результата пользователю

#### 8a. В content.js (индикатор на странице)

```javascript
function displayResult(response) {
  // Удаляем предыдущий индикатор, если есть
  const existing = document.getElementById('phishbuster-indicator');
  if (existing) existing.remove();

  // Создаём плавающий индикатор
  const indicator = document.createElement('div');
  indicator.id = 'phishbuster-indicator';
  indicator.style.cssText = `
    position: fixed;
    top: 10px;
    right: 10px;
    z-index: 999999;
    padding: 8px 16px;
    border-radius: 8px;
    font-family: sans-serif;
    font-size: 14px;
    font-weight: bold;
    color: white;
    background: ${getVerdictColor(response.verdict)};
    box-shadow: 0 2px 8px rgba(0,0,0,0.3);
    cursor: pointer;
    transition: opacity 0.3s;
  `;
  indicator.textContent = getVerdictText(response.verdict);
  
  indicator.addEventListener('click', () => {
    indicator.style.opacity = '0';
    setTimeout(() => indicator.remove(), 300);
  });
  
  document.body.appendChild(indicator);
  
  // Автоскрытие через 5 секунд
  setTimeout(() => {
    if (indicator.parentNode) {
      indicator.style.opacity = '0';
      setTimeout(() => { if (indicator.parentNode) indicator.remove(); }, 300);
    }
  }, 5000);
}
```

#### 8b. В popup.js (детальный просмотр)

```javascript
// popup.js запрашивает данные при открытии
chrome.tabs.query({ active: true, currentWindow: true }, (tabs) => {
  const tabId = tabs[0].id;
  
  chrome.runtime.sendMessage(
    { action: "getResult", tabId: tabId },
    (response) => {
      if (response) {
        updatePopupUI(response);
      }
    }
  );
});
```

## Временная диаграмма

```
Время:  0ms    200ms    500ms    600ms    700ms    750ms    800ms
        │       │        │        │        │        │        │
Сайт:   [загрузка DOM.....]      │        │        │        │
        │       │        │        │        │        │        │
Chrome: │   content.js    │        │        │        │        │
        │   внедрён       │        │        │        │        │
        │       │        │        │        │        │        │
Content:│    [извлечение метрик..]→sendMsg→│        │        │
        │       │        │        │        │        │        │
Backgrd:│       │        │      [получ.→загрузка   [скоринг]→кэш→отправка
        │       │        │        │      rules.json] │        │
        │       │        │        │        │        │        │
Content:│       │        │        │        │        │   [индикатор на странице]
        │       │        │        │        │        │        │
Popup:  │       │        │        │        │        │        │
(если   │       │        │        │        │        │        │
открыт) │       │        │        │        │        │     [запрос→получение→отображение]
```

**Итого**: от загрузки страницы до отображения результата — **~300-800 мс**, что незаметно для пользователя.
