## Подробное описание компонентов

### 1. manifest.json — Конфигурация расширения

**Назначение**: Метаданные расширения, декларация разрешений, регистрация скриптов.

**Входные данные**: Нет (статический файл).

**Выходные данные**: Конфигурация для Chrome при загрузке расширения.

**Взаимодействие**: Читается браузером при установке расширения.

**Используемые API**: Нет (конфигурационный файл).

```json
{
  "manifest_version": 3,
  "name": "PhishBuster RU",
  "version": "1.0.0",
  "description": "Браузерное расширение для выявления фишинговых сайтов на основе анализа цифровых метрик",
  "permissions": [
    "storage",
    "activeTab",
    "scripting"
  ],
  "host_permissions": [
    "<all_urls>"
  ],
  "background": {
    "service_worker": "background.js"
  },
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["content.js"],
      "run_at": "document_idle"
    }
  ],
  "action": {
    "default_popup": "popup.html",
    "default_icon": {
      "16": "icons/icon16.png",
      "48": "icons/icon48.png",
      "128": "icons/icon128.png"
    }
  },
  "icons": {
    "16": "icons/icon16.png",
    "48": "icons/icon48.png",
    "128": "icons/icon128.png"
  }
}
```

**Обоснование разрешений**:
| Разрешение | Зачем |
|---|---|
| `storage` | Доступ к `chrome.storage.local` для кэширования результатов |
| `activeTab` | Доступ к текущей вкладке для получения URL |
| `scripting` | Возможность программно внедрять content script |
| `<all_urls>` (host_permissions) | Анализ DOM на любой веб-странице |

---

### 2. content.js — Анализатор DOM

**Назначение**: Извлечение метрик из DOM загруженной страницы и отправка данных в background.js.

**Входные данные**:
- DOM-дерево текущей страницы
- URL текущей страницы
- `<title>` страницы

**Выходные данные**:
- Объект с извлечёнными метриками (селекторы, ключевые слова, домен)
- Отправляется через `chrome.runtime.sendMessage()`

**Взаимодействие**:
- → `background.js`: отправляет извлечённые метрики
- ← `background.js`: получает результат анализа (score + verdict)
- Использует `MutationObserver` для отслеживания изменений DOM

**Используемые API**:
- `document.querySelectorAll()` — поиск DOM-элементов
- `document.title` — получение заголовка
- `window.location.href` — получение URL
- `MutationObserver` — отслеживание изменений DOM
- `chrome.runtime.sendMessage()` — обмен сообщениями
- `chrome.runtime.onMessage.addListener()` — приём сообщений

---

### 3. background.js — Ядро скоринга (Service Worker)

**Назначение**: Загрузка базы правил, вычисление риска, кэширование результатов, координация компонентов.

**Входные данные**:
- Метрики от content.js (селекторы, ключевые слова, домен)
- Запросы от popup.js (получить результат текущей вкладки)

**Выходные данные**:
- Итоговый score (0-100) и вердикт (safe / warning / danger)
- Список сработавших правил
- Сохранение в кэш

**Взаимодействие**:
- ← `content.js`: получает метрики через `chrome.runtime.onMessage`
- → `content.js`: отправляет результат анализа
- ← `popup.js`: получает запрос на данные текущей вкладки
- → `popup.js`: отправляет кэшированный результат
- ↔ `rules.json`: загружает базу правил при инициализации
- ↔ `chrome.storage.local`: читает/пишет кэш

**Используемые API**:
- `chrome.runtime.onMessage.addListener()` — приём сообщений
- `chrome.runtime.sendMessage()` — отправка сообщений
- `chrome.storage.local.get()` / `set()` — работа с кэшем
- `fetch('rules.json')` — загрузка базы правил


---

### 4. popup.html + popup.js — Пользовательский интерфейс

**Назначение**: Отображение результатов анализа пользователю.

**Входные данные**:
- Кэшированный результат анализа от background.js

**Выходные данные**:
- Визуальное представление безопасности текущей страницы

**Взаимодействие**:
- → `background.js`: запрашивает данные через `chrome.runtime.sendMessage()`
- ← `background.js`: получает score, verdict, причины, версию базы

**Используемые API**:
- `chrome.runtime.sendMessage()` — обмен сообщениями
- `chrome.tabs.query()` — получение активной вкладки
- Стандартный DOM API для обновления UI

---

### 5. rules.json — База эталонных правил

**Назначение**: Хранение эталонных признаков легитимных сайтов для сравнения.

**Входные данные**: Нет (статические данные).

**Выходные данные**: Правила анализа для background.js.

**Взаимодействие**: Читается background.js при инициализации через `fetch()`.

**Используемые API**: Нет (файл данных).

## Диаграмма взаимодействия компонентов

```
Пользователь открывает сайт
            │
            ▼
    ┌───────────────┐
    │  Chrome       │
    │  загружает    │
    │  content.js   │
    └───────┬───────┘
            │
            ▼
    ┌───────────────────────┐
    │ content.js извлекает: │
    │ • URL (window.loc.)   │
    │ • Title (document.)   │
    │ • DOM-селекторы       │
    │ • Формы ввода         │
    └───────┬───────────────┘
            │ chrome.runtime.sendMessage(metrics)
            ▼
    ┌───────────────────────┐
    │ background.js:        │
    │ 1. Загружает rules    │
    │ 2. Вычисляет score    │
    │ 3. Кэширует результат │
    └───────┬───────────────┘
            │ chrome.runtime.sendMessage(result)
            ▼
    ┌───────────────────────┐
    │ content.js:           │
    │ Показывает badge/     │
    │ индикатор на странице │
    └───────────────────────┘
    
Пользователь кликает на иконку расширения
            │
            ▼
    ┌───────────────────────┐
    │ popup.js:             │
    │ Запрашивает данные    │
    │ из background.js      │
    └───────┬───────────────┘
            │ chrome.runtime.sendMessage({action: "getResult"})
            ▼
    ┌───────────────────────┐
    │ background.js:        │
    │ Возвращает            │
    │ кэшированный результат│
    └───────┬───────────────┘
            │
            ▼
    ┌───────────────────────┐
    │ popup.html:           │
    │ Отображает:           │
    │ • Score               │
    │ • Вердикт             │
    │ • Причины             │
    │ • Версию базы         │
    └───────────────────────┘
```
