# Раздел 2: Демонстрационные тестовые страницы

## Принцип создания безопасных тестовых страниц

**Правило**: мы НЕ создаём реалистичные копии существующих сервисов. Вместо этого используем **учебные абстрактные страницы** с достаточным количеством признаков для срабатывания алгоритма, но без воспроизведения брендинга реальных организаций.

Все страницы используют вымышленные названия: «TestBank», «DemoService», «SamplePortal» и т.д.

---

### 1. testbank-fake-login.html

**Назначение**: Имитация фишинговой страницы с поддельным доменом, формой входа и urgency-текстом.

**Признаки срабатывания**:
- Селекторы: форма входа, поле пароля, кнопка «Войти»
- Ключевые слова: «личный кабинет», «вход в систему»
- Urgency: «срочно», «заблокирован»
- Контекст: файл открыт локально, URL не совпадает с legitimate

**Ожидаемый результат**: `danger` (score ≥ 60)

**Примечание**: Поскольку страница открывается по `file://`, домен не совпадёт ни с одним suspiciousPattern, но если мы добавим тестовые ключевые слова, совпадающие с keywords из rules.json, и форму входа — score будет зависеть от selector + keyword = 35 + 25 = 60 → **попадёт в danger**.

> ⚠️ **Важно**: для корректного срабатывания на `file://` нужно, чтобы keywords совпадали с keywords сервиса из rules.json. Для демонстрации можно временно добавить «TestBank» в titleKeywords или использовать слова, которые уже есть в базе.

## Как открывать тестовые страницы

### Способ 1: `file://` (проще для демонстрации)

1. Открыть Chrome
2. `Ctrl+O` → выбрать HTML-файл из папки `tests/`
3. Расширение проанализирует страницу

**Ограничения `file://`**:
- Домен = пустой или `null` → доменная метрика не сработает
- Content script может не внедриться, если в `manifest.json` не разрешён доступ к `file://`
- **Решение**: в `manifest.json` добавить `"file://"` в `host_permissions` (уже есть `<all_urls>`)

**Когда использовать**: для быстрой демонстрации keyword и selector-метрик.

### Способ 2: Локальный сервер (рекомендуется для полной демонстрации)

```bash
# Python 3 (есть на большинстве компьютеров)
cd phishbuster
python3 -m http.server 8080
```

Затем открыть: `http://localhost:8080/tests/danger/testbank-fake-login.html`

**Преимущества**:
- Content script гарантированно внедряется
- URL выглядит как настоящий сайт
- Можно задать кастомный домен через hosts-файл

**Когда использовать**: для полной демонстрации всех трёх метрик.

### Способ 3: Кастомный домен через hosts-файл (для продвинутой демонстрации)

1. Открыть `C:\Windows\System32\drivers\etc\hosts` (Windows) или `/etc/hosts` (Linux/Mac)
2. Добавить: `127.0.0.1 fake-sberbank.ru`
3. Запустить локальный сервер
4. Открыть: `http://fake-sberbank.ru:8080/tests/danger/testbank-fake-login.html`

**Преимущества**: демонстрирует работу доменной метрики на «подозрительном» домене.
**Когда использовать**: если комиссия попросит показать доменную метрику в действии.
