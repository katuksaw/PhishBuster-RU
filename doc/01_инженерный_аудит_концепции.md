# PhishBuster RU — Инженерный аудит концепции проекта

## 1. Выбор платформы: Chrome Extension (Manifest V3)

### Зачем используется
Браузерное расширение — единственный способ получить программный доступ к DOM-дереву открытой веб-страницы, URL-адресу и заголовкам непосредственно в момент загрузки сайта, не требуя от пользователя установки отдельного приложения.

### Почему подходит для школьного проекта
- **Низкий порог входа**: JavaScript — изучается в школьном курсе информатики
- **Мгновенный результат**: расширение работает сразу, без развёртывания сервера
- **Реальная польза**: ученик создаёт работающий продукт, а не учебный пример
- **Широкая аудитория**: Chrome занимает ~65% рынка браузеров (2024-2025)

### Альтернативы
| Альтернатива | Плюсы | Минусы |
|---|---|---|
| Firefox Add-on | Аналогичный API | Меньшая аудитория (~3%) |
| Серверный прокси | Полный контроль трафика | Требует сервер, сложнее развёртывание |
| Отдельное desktop-приложение | Не зависит от браузера | Сложнее, не интегрируется с браузером |
| Пользовательский скрипт (Tampermonkey) | Быстрый прототип | Зависит от стороннего расширения, не distribution-ready |

### Почему выбран Chrome Extension Manifest V3
- Стандарт индустрии для расширений (Manifest V2 официально устарел)
- Manifest V3 — актуальный стандарт, обязательный для публикации в Chrome Web Store с 2024 года
- Service Worker вместо background page — более энергоэффективный
- Декларативные правила безопасности через `declarativeNetRequest`
- Именно V3 покажет комиссии, что ученик ориентируется в актуальных технологиях

## 2. Язык программирования: JavaScript (ES2022+)

### Зачем используется
JavaScript — единственный язык, который выполняется нативно в браузерных расширениях. ES2022+ предоставляет современные конструкции (arrow functions, destructuring, async/await, optional chaining), делающие код читаемым и лаконичным.

### Почему подходит для школьного проекта
- Соответствует школьной программе по информатике (алгоритмы, функции, массивы)
- Не требует компиляции — работает «из коробки»
- Огромное количество учебных материалов на русском языке

### Альтернативы
| Альтернатива | Плюсы | Минусы |
|---|---|---|
| TypeScript | Статическая типизация, меньше ошибок | Требует сборку (tsc), усложняет проект |
| WebAssembly (Rust/C++) | Максимальная производительность | Неоправданно сложно для школьного проекта |
| Python (через Brython) | Знаком ученикам | Низкая производительность, нестандартно |

### Почему выбран JavaScript без TypeScript
- Для комиссии важна понятность кода — без слоя абстракций
- Проект не достигает масштаба, где типизация критична (~500 строк кода)
- Отсутствие build-step упрощает демонстрацию: открыл файл → показал код → работает

## 3. Формат базы правил: JSON

### Зачем используется
JSON — текстовый формат хранения структурированных данных, нативно поддерживаемый JavaScript через `JSON.parse()` / `JSON.stringify()`.

### Почему подходит для проекта
- Читаемый человеком — можно открыть в любом текстовом редакторе
- Не требует установки СУБД или сервера
- Понятен комиссии: структура данных видна визуально

### Альтернативы
| Альтернатива | Плюсы | Минусы |
|---|---|---|
| SQLite (через WebAssembly) | Запросы, индексы | Избыточно для ~20 записей |
| CSV | Простота | Нет вложенной структуры |
| Удалённая база (API) | Актуальность | Требует сервер, интернет-зависимость |

### Почему выбран JSON
- Объём базы — 5-20 сервисов, каждый с ~10 полями → ~5 КБ данных
- JSON нативно встроен в JS, не требует зависимостей
- Возможность версионирования (`"version": "1.2.0"`) — демонстрирует инженерный подход

## 4. Архитектура с разделением на content.js / background.js / popup.js

### Зачем используется
Manifest V3 строго разделяет контексты выполнения:
- **Content script** — работает в контексте веб-страницы, имеет доступ к DOM
- **Service worker** — фоновый скрипт, имеет доступ к `chrome.*` API
- **Popup** — интерфейс расширения, отдельная HTML-страница

### Почему подходит для проекта
- Демонстрирует понимание модульной архитектуры
- Каждый модуль отвечает за одну задачу (Single Responsibility Principle)
- Позволяет объяснить комиссии «кто что делает» простыми словами

### Альтернативы
| Альтернатива | Плюсы | Минусы |
|---|---|---|
| Один монолитный content script | Простота | Невозможно в Manifest V3 (нет доступа к chrome.storage из content script напрямую в некоторых сценариях) |
| Внедрение через `chrome.scripting.executeScript` | Динамическое внедрение | Сложнее отлаживать |

### Почему выбрано разделение
- Это **единственный корректный** способ в Manifest V3
- Разделение демонстрирует знание браузерной архитектуры
- Позволяет одновременно показывать popup и анализировать страницу
